## SQLite Schema

```sql
-- Item cache (TTL-based)
CREATE TABLE cache (
    key         TEXT PRIMARY KEY,     -- "csfloat:listing:12345" or "steam:price:AK-47_Redline_FT"
    value       TEXT NOT NULL,        -- JSON serialized data
    created_at  TEXT NOT NULL,        -- ISO 8601 timestamp
    expires_at  TEXT NOT NULL,        -- ISO 8601 timestamp (created_at + TTL)
    source      TEXT NOT NULL         -- "csfloat" | "steam"
);
CREATE INDEX idx_cache_expires ON cache(expires_at);

-- Price history (append-only)
CREATE TABLE price_history (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    weapon      TEXT NOT NULL,
    skin        TEXT NOT NULL,
    quality     TEXT NOT NULL,
    stattrak    INTEGER NOT NULL DEFAULT 0,  -- boolean
    price       REAL NOT NULL,
    volume      INTEGER,                      -- daily volume at time of recording
    source      TEXT NOT NULL,                -- "csfloat" | "steam"
    recorded_at TEXT NOT NULL                 -- ISO 8601 timestamp
);
CREATE INDEX idx_price_history_item ON price_history(weapon, skin, quality, stattrak);
CREATE INDEX idx_price_history_time ON price_history(recorded_at);

-- Decision log (append-only, labeled data for future ML)
CREATE TABLE decision_log (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp       TEXT NOT NULL,           -- ISO 8601
    input_url       TEXT,                    -- original input URL/ID
    canonical_json  TEXT NOT NULL,           -- CanonicalItem as JSON
    instance_json   TEXT,                    -- ExactInstance as JSON (null if degraded)
    pricing_json    TEXT NOT NULL,           -- PricingResult as JSON
    liquidity_json  TEXT NOT NULL,           -- LiquidityResult as JSON
    decision_json   TEXT NOT NULL,           -- Decision as JSON
    action          TEXT NOT NULL,           -- "buy" | "no_buy" | "review"
    confidence      REAL NOT NULL,           -- 0.0-1.0
    listing_price   REAL NOT NULL,
    estimated_value REAL NOT NULL,
    user_override   TEXT,                    -- manual correction: "actually_good" | "actually_bad" | null
    notes           TEXT                     -- free-form user annotation
);
CREATE INDEX idx_decision_log_time ON decision_log(timestamp);
CREATE INDEX idx_decision_log_action ON decision_log(action);

-- Sticker price cache (refreshed daily)
CREATE TABLE sticker_prices (
    name        TEXT PRIMARY KEY,
    price       REAL NOT NULL,
    updated_at  TEXT NOT NULL              -- ISO 8601
);
```

**Cache TTL Policy:**
| Data Type | TTL | Rationale |
|-----------|-----|-----------|
| Market prices | 1 hour | Prices change frequently |
| Listing data | 15 min | Listings can sell quickly |
| Item identity | 7 days | Canonical items don't change |
| Sticker prices | 24 hours | Sticker market moves slowly |
| Float/pattern data | 30 days | Instance attributes are immutable |

### 6. Pricing Engine (`engine/pricing.py`)

**Input:** `CanonicalItem` + `ExactInstance` + market data + `Settings`
**Output:** `PricingResult`

**Steps:**
```
1. Fetch base_price:
   - Query price_history for canonical item
   - Median of last 50 sales (or all if <50, min 5)
   - StatTrak items use separate bucket

2. Classify item:
   - Check float percentile within wear range
   - Check sticker values
   - Check pattern DB
   - → ItemClass.COMMODITY or ItemClass.EXACT_PREMIUM

3. Calculate premium (if exact_premium):
   float_premium   = base_price × float_factor(percentile)
   sticker_premium = sum(sticker_price × mult × position_bonus × condition)
   pattern_premium = special_patterns.json lookup
   stattrak_premium = base_price × 0.15 (if ST + float < 0.01)

4. estimated_value = base_price + float_premium + sticker_premium + pattern_premium + stattrak_premium
```

**Float Factor Table:**
| Percentile | Factor |
|-----------|--------|
| Top 1% | 0.50 |
| Top 5% | 0.20 |
| Top 10% | 0.10 |
| Bottom 5% | -0.10 |
| Middle 80% | 0.00 |

**Wear Tier Ranges:**
| Quality | Float Range |
|---------|------------|
| Factory New | 0.00 - 0.07 |
| Minimal Wear | 0.07 - 0.15 |
| Field-Tested | 0.15 - 0.38 |
| Well-Worn | 0.38 - 0.45 |
| Battle-Scarred | 0.45 - 1.00 |

### 7. Liquidity Analyzer (`engine/liquidity.py`)

**Input:** `CanonicalItem` + market data + `Settings`
**Output:** `LiquidityResult`

**Algorithm:**
```
1. avg_daily_volume = total_sales_30d / 30
2. avg_spread_pct = (ask - bid) / midpoint × 100
3. grade:
   - HIGH: avg_daily_volume > 10
   - MEDIUM: 1 <= avg_daily_volume <= 10
   - LOW: avg_daily_volume < 1
   - UNKNOWN: insufficient data (<5 sales in 30d)
4. time_to_sell:
   - HIGH: min=0, max=1 (days)
   - MEDIUM: min=1, max=7
   - LOW: min=7, max=30
   - UNKNOWN: min=0, max=999
5. safe_exit_price = min(estimated_value × 0.85, p10(comparable_sales))
   floor = base_price × 0.90
```

### 8. Decision Engine (`engine/decision.py`)

**Input:** `PricingResult` + `LiquidityResult` + listing_price + `Settings`
**Output:** `Decision`

**Rule-Based Logic:**
```
margin_pct = (estimated_value - listing_price) / listing_price × 100
safe_margin = (safe_exit_price - listing_price) / listing_price × 100

confidence = weighted_avg(
    data_completeness × 0.3,
    price_data_quality × 0.25,
    sigmoid(margin_pct / 20%) × 0.25,
    liquidity_score × 0.2,
)

# Incomplete data → cap at 0.49 (triggers REVIEW)
if any_incomplete_flag:
    confidence = min(confidence, 0.49)

# Decision rules
if confidence < 0.5:
    action = REVIEW
elif margin_pct > 15 and safe_margin > 0 and liquidity.grade != LOW:
    action = BUY
elif margin_pct < -5:
    action = NO_BUY
elif liquidity.grade == UNKNOWN:
    action = REVIEW
else:
    action = REVIEW  # borderline → conservative
```

### 9. Pipeline Orchestrator (`pipeline.py`)

**Input:** URL or manual item spec
**Output:** `Decision` + all intermediate results

**Flow:**
1. Parse input (URL → source + ID, or manual → CanonicalItem)
2. Check cache for all stages
3. Run pipeline stages in sequence
4. Log decision
5. Return Decision

**Error Propagation:**
```
Source fail (after retries) → check cache
  ├── cache hit → continue with stale data, flag degraded
  └── cache miss → hard fail with "cannot fetch data"

Identity fail → hard fail (can't proceed)

Enrichment fail → warn, continue as commodity (degraded)

Pricing fail → return base_price only, flag incomplete

Liquidity fail → set UNKNOWN, flag risk
```
